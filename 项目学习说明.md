# 中小饭店点餐系统学习版说明

这个版本把原来集中在 `app.cpp` 里的业务代码拆成多个文件。阅读时不要把它当成“很多文件堆在一起”，而要把它当成一条从底层到上层的学习链路：

1. 先认识数据怎么存。
2. 再认识数据怎么连。
3. 再认识数据怎么改。
4. 再认识数据怎么存盘。
5. 最后再看页面怎么把这些规则串起来。

## 文件分工

- `main.cpp`：程序入口。你只需要看它怎么初始化 UI，然后把控制权交给主菜单。
- `ui.hpp` / `ui.cpp`：控制台界面工具。这里不关心业务，只关心“怎么显示”“怎么输入”“怎么确认”。
- `app.hpp`：对外公开的业务页面声明。它像一张菜单目录，告诉外部可以调用哪些页面。
- `app_internal.hpp`：内部说明书。这里放数据结构、状态常量、全局变量和内部函数声明，是理解整个系统的起点。
- `app.cpp`：模块导读文件。它的存在是为了把学习路线说清楚，而不是承担具体业务。
- `app_core.cpp`：核心规则层。这里决定“链表怎么建、怎么找、怎么删、怎么改、订单金额怎么重算、库存怎么同步”。
- `app_persistence.cpp`：持久化层。这里决定“内存里的链表怎么写进文件”“文件里的文本怎么恢复成节点”。
- `app_display.cpp`：展示层。这里主要是把链表内容打印成表格、小票、列表。
- `app_menus.cpp`：导航层。这里决定“系统从哪里进入”“管理员和服务员分别怎么跳转”。
- `app_admin.cpp`：管理员业务层。这里负责基础资料维护、查询、统计和系统设置。
- `app_waiter.cpp`：服务员业务层。这里负责开台、点餐、改单、退菜、结账。
- `linked_list_demo.cpp`：链表入门示例。它不属于主系统业务，但它是理解 `app_internal.hpp` 和 `app_core.cpp` 的最好起点。

## 推荐阅读顺序

1. 先看 `linked_list_demo.cpp`。
   这一步的目标不是记代码，而是建立最小概念模型。你要看懂“一个节点是什么”“头指针是什么”“next 为什么能把数据串起来”“删除时为什么需要前一个节点”。

2. 再看 `app_internal.hpp`。
   这里开始进入真实项目的数据结构。你要把 `UserNode`、`DishNode`、`TableNode`、`OrderNode`、`OrderItemNode` 当成五种业务实体来理解，而不是把它们当成一堆字段。

3. 接着看 `app_core.cpp`。
   这一层是系统的“规则中心”。你不用一口气看完所有函数，但一定要优先看清楚：
   - 节点是怎么创建的
   - 节点是怎么追加到链表尾部的
   - 节点是怎么查找的
   - 订单金额是怎么重算的
   - 点菜、退菜、取消订单时库存和销量是怎么变化的

4. 然后看 `app_persistence.cpp`。
   这一层负责把数据保留下来。你要理解“链表里的节点”与“txt 文件里的一行文本”之间的对应关系，也要理解为什么有些字段必须保留原始值，比如订单中的菜名、单价和时间。

5. 再看 `app_menus.cpp`。
   这一层负责“把人带到正确的地方”。你要看懂主菜单如何把用户分流到管理员和服务员两条路径，以及登录成功后为什么会保存当前账号信息。

6. 最后看 `app_admin.cpp` 和 `app_waiter.cpp`。
   这两份文件就是具体业务流程的入口。它们会不断调用底层规则层的函数，所以最适合在你已经理解链表和订单规则之后再看。

## 学习时要注意

1. 不要先死记函数名，先看“数据流向”。比如用户点餐后，数据是怎样从菜品链表、订单链表、餐桌链表一起变化的。
2. 不要把一个功能拆得太碎地看。像“开台点餐”这种流程，应该当成一整条业务线来看，而不是只看 `showOpenTableOrder()` 这一行。
3. `app_core.cpp` 是最值得反复看的地方，因为这里藏着所有“正确性”问题。只要库存、金额、餐桌状态有一个不同步，系统就会出错。
4. `app_persistence.cpp` 里要特别注意“保存的是什么”。有些字段保存的是实时状态，有些字段保存的是历史记录，二者不能混。
5. `OrderNode` 和 `OrderItemNode` 是一对主从关系。订单是“单”，明细是“条目”。先看懂这层关系，再看结账和查询会顺很多。
6. `TableNode.activeOrderId` 是很关键的桥梁字段。它把餐桌和当前订单连起来，很多流程都靠它找回订单。
7. 你看到 `g_` 前缀时，要立刻想到“这是全局状态”，例如当前登录账号、链表头指针、加载状态、下一个订单号。
8. 如果你对链表还不熟，先不要急着啃管理员管理模块，先把 `linked_list_demo.cpp` 读透。它会帮你减少很多理解成本。

## 读代码时建议盯住的几个问题

1. 这个函数是在“查数据”，还是在“改数据”？
2. 它改的是哪条链表？用户、菜品、餐桌，还是订单？
3. 如果订单金额变了，餐桌上的当前金额有没有一起变？
4. 如果菜品数量减少了，库存有没有恢复？
5. 如果订单为空了，餐桌有没有被释放？
6. 如果数据改了，最后有没有调用 `saveAll()`？

这些问题你每看一个函数都问一遍，理解会快很多。

## 编译方式

双击 `build.bat`，或在当前目录运行：

```bat
build.bat
```

也可以手动编译：

```bat
g++ -std=c++11 -Wall -Wextra -finput-charset=UTF-8 -fexec-charset=UTF-8 main.cpp ui.cpp app.cpp app_core.cpp app_persistence.cpp app_display.cpp app_menus.cpp app_admin.cpp app_waiter.cpp -o restaurant.exe
```

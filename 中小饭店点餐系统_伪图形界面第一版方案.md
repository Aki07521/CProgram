# 中小饭店点餐管理系统伪图形界面第一版方案

> 适用场景：高级程序设计语言课程设计第一版界面框架  
> 推荐语言：C++ 为主，保留 C 风格结构体和链表思想  
> 图形方案：Windows 控制台伪图形界面  
> 第三方库：不使用 EasyX、Qt、MFC 等第三方库

---

## 1. 方案定位

本方案用于完成一个 **“中小饭店点餐管理系统”** 的第一版界面框架。

第一版不直接实现完整点餐业务逻辑，而是先完成：

- 主菜单
- 管理员登录
- 服务员登录
- 管理员功能菜单
- 服务员功能菜单
- 菜品表格展示
- 餐桌状态展示
- 结账小票展示
- 统一提示页面
- 上下键选择菜单
- 彩色高亮
- ASCII 边框伪图形界面

后续第二版再接入：

- 链表数据结构
- 文件存储
- 用户管理
- 菜品增删改查
- 餐桌管理
- 点餐管理
- 退菜、改菜
- 结账
- 营收统计

## 2. 技术路线

### 2.1 推荐使用 C++，但不要过度复杂化

课程设计允许使用 C 或 C++。本方案建议使用 **C++**，因为 C++ 在输入输出、字符串处理和代码组织方面更方便。

可以使用：

```cpp
#include <iostream>
#include <iomanip>
#include <string>
#include <vector>
#include <windows.h>
#include <conio.h>
```

但是需要注意：

> 课程设计中涉及增删改查的系统，数据结构建议仍然采用链表实现。  
> 所以正式业务数据不要完全依赖 `vector` 存储，后续菜品、餐桌、订单、用户等核心数据建议使用链表。

第一版界面菜单项可以临时使用 `vector<string>`，因为它只是界面选项，不是正式业务数据。

---

## 3. 为什么选择控制台伪图形界面

### 3.1 不依赖第三方库

不使用 EasyX、Qt、MFC 等库，降低环境配置难度。

### 3.2 运行稳定

只要是 Windows 环境，配合常见 C++ 编译器基本都可以运行。

### 3.3 适合录制运行视频

菜单、表格、小票、颜色、高亮选择都能体现系统完成度。

### 3.4 后续容易扩展

第一版只写界面框架，后续可以把静态表格替换成链表遍历输出，把登录演示替换成用户文件验证。

---

## 4. 第一版实现目标

第一版需要实现：

1. 程序启动后显示主菜单。
2. 菜单支持上下键选择。
3. 当前选中项高亮显示。
4. 按 Enter 确认选择。
5. 按 Esc 返回上一级。
6. 管理员和服务员分别登录。
7. 登录成功后进入不同角色菜单。
8. 管理员和服务员看到的菜单不同。
9. 菜品管理页面可以展示静态菜品表格。
10. 餐桌管理页面可以展示静态餐桌状态表格。
11. 顾客结账页面可以展示静态结账小票。
12. 暂未完成的功能显示统一提示页面。
13. 采用多文件结构。
14. 代码有清晰注释。
15. 后续方便接入链表和文件存储。

### 20.2 中文乱码处理

如果出现中文乱码，可以尝试：

1. 将源文件保存为 ANSI 或 GBK 编码。
2. 在代码中加入：

```cpp
system("chcp 936 > nul");
```

3. 或者如果环境支持 UTF-8，可以使用：

```cpp
system("chcp 65001 > nul");
```

不同电脑环境不一样，选择一种稳定显示中文的方式即可。

## 23. 最终建议

第一版不要追求功能完整，重点是把界面框架搭好。

1. 先做 UI 基础函数。

4. 再做两个角色菜单。
